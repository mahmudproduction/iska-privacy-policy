Политика конфиденциальности — I.S.K.A

English version is available below.

Дата обновления: 2025-09-24

Кратко

Расширение не собирает, не хранит и не передает персональные данные пользователей.

Какие данные обрабатываются

  • Расширение использует API браузера chrome.storage для локального хранения настроек, необходимых для работы расширения. Эти данные остаются на устройстве пользователя.
  • Расширение может временно анализировать содержимое активной вкладки через chrome.scripting исключительно для выполнения функций расширения. Содержимое страниц не сохраняется и не передается третьим лицам.

Что не делается

  • Не собираем контактные данные, учетные записи, пароли, финансовые, медицинские и иные чувствительные сведения.
  • Не ведем аналитику использования, не используем куки и сторонние трекеры.
  • Не продаем и не передаем пользовательские данные третьим лицам.

Разрешения Chrome

  • storage — хранение настроек расширения локально.
  • tabs — работа с активной вкладкой по явному действию пользователя.
  • scripting — безопасное выполнение встроенных скриптов на странице для реализации функций.
  • clipboardWrite — копирование текста в буфер обмена по запросу пользователя.

Доступ к хостам

Если в манифесте указаны разрешенные хосты, обращения к ним выполняются только по HTTPS и исключительно для получения данных, необходимых для работы расширения. Внешние скрипты не загружаются и не исполняются.

Безопасность

Код расширения поставляется в составе пакета из Chrome Web Store и не выполняет удаленный код. Данные, сохраняемые в chrome.storage, остаются на устройстве пользователя.

Privacy Policy — I.S.K.A (Chrome Extension)

Last updated: 2025-09-24

Summary

This extension does not collect, store, or transmit any personally identifiable information.

Data Processing

  • The extension uses chrome.storage to keep user settings locally on the device.
  • Page content may be temporarily accessed via chrome.scripting solely to provide the stated functionality. Page content is neither stored nor shared.

We do not do

  • We do not collect contact details, accounts, passwords, financial, medical, or other sensitive information.
  • No usage analytics, cookies, or third‑party trackers.
  • No selling or sharing of user data with third parties.

Chrome Permissions

  • storage — keep extension settings locally.
  • tabs — interact with the active tab upon user action.
  • scripting — execute built‑in scripts on pages to provide features.
  • clipboardWrite — copy text to clipboard when requested by the user.

Host Access

If host permissions are declared, requests are made via HTTPS only and strictly to obtain data required for the core functionality. No remote scripts are loaded or executed.

Security

The extension ships as a packaged item from the Chrome Web Store and does not execute remote code. Data stored via chrome.storage remains on the user's device.

Changes

We may update this document from time to time. The latest version supersedes previous versions once published at this address.
